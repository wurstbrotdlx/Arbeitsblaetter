So baust du dein Spiel
mit ChatGPT

Du sagst der KI in normaler Sprache, was du willst. Sie schreibt den Code. Hier steht Schritt für Schritt, wie das gut klappt.

↗ChatGPT öffnen ▸Zum Editor

Zuerst: drei Regeln

1
Keine echten Namen, keine privaten Daten.

Nur Spielideen in die KI tippen. Keine Adressen, keine Handynummern, nichts Persönliches von dir oder anderen.

2
Fehler sind normal.

Wenn etwas nicht geht, ist das kein Problem, sondern dein eigentlicher Job. Fehler zu beschreiben und zurückzumelden ist die wichtigste Fähigkeit hier.

3
Immer nur eine Sache auf einmal ändern.

Eine Änderung, dann testen, dann die nächste. So merkst du sofort, was gewirkt hat und was den Fehler gemacht hat.

Der Ablauf in 6 Schritten

Tipp auf einen Schritt, um ihn aufzuklappen.

Überleg dir in einem Satz, was passieren soll. Ein Fang-Spiel? Ein Quiz? Memory? Ein Klick-Marathon? Je klarer deine Idee, desto leichter geht der Rest.

TippFang klein an. „Ein Ball fällt runter, ich klicke ihn an und bekomme einen Punkt“ ist schon ein komplettes Spiel.

Öffne ChatGPT und beschreib dein Spiel so genau wie möglich. Die KI kann nicht raten, was in deinem Kopf ist. Sie baut das, was du sagst.

Unten im Prompt-Baukasten kannst du dir mit ein paar Klicks eine fertige, gute Ansage zusammenstellen und kopieren.

MerkeSag der KI immer dazu: „Alles in eine HTML-Datei, ich bin Schüler, halte den Code einfach.“

ChatGPT schreibt dir jetzt HTML-Code. So bringst du ihn ins Spiel:

1. Den ganzen Code markieren und kopieren.
2. In den Editor wechseln.
3. Alles löschen, was im Code-Feld steht, und deinen Code einfügen.
4. Auf Starten drücken.

TippChatGPT zeigt Code oft in einem grauen Kasten mit einem kleinen „Kopieren“-Knopf oben rechts. Nimm den, dann hast du sicher alles.

Schau dir dein Spiel an und probier es aus. Klick alles durch. Stell dir dabei zwei Fragen: Macht es, was ich wollte? Und: Was fehlt oder stört noch?

Merk dir, was noch nicht passt. Genau das sagst du im nächsten Schritt der KI.

Etwas geht nicht? Perfekt. Genau das ist die eigentliche Aufgabe. Der Trick: Sag der KI ganz genau, was schiefläuft.

Nicht: „geht nicht“.
Sondern: „Der Ball bewegt sich nicht, wenn ich klicke.“

Unten hilft dir der Fehler-Melder, das gut zu formulieren und zu kopieren.

WichtigJe genauer du das Problem beschreibst, desto genauer kann die KI es reparieren. „Beschreiben können“ ist hier die Superkraft.

Jetzt machst du dein Spiel besser: eine Idee dazu prompten, neuen Code holen, testen. Immer nur eine Änderung, dann wieder testen.

Wenn du zufrieden bist: im Editor auf Speichern tippen (du bekommst eine .html-Datei) und diese über itslearning abgeben.

TippNimm dir am Ende die Checkliste ganz unten vor, dann vergisst du nichts.

Schwammig oder konkret?

Dieselbe Idee, zwei Ansagen an die KI. Sieh den Unterschied.

✗ So besser nicht
„Mach mir ein cooles Spiel.“
Die KI muss raten. Ergebnis meistens irgendwas, aber selten das, was du im Kopf hattest.
✓ So klappt es
„Baue ein Fang-Spiel: ein blauer Kreis fällt von oben, ich klicke ihn an und bekomme pro Treffer einen Punkt.“
Klar, konkret, testbar. Die KI weiß genau, was sie bauen soll.
